##### 版本号并发控制

1. 主副同步：

shard与replica同步，并发同步，基于_version来保证数据的正确，如果后版本号先到，前版本号后到，前版本号的数据将被丢弃

2. 内部version

```curl
PUT /shop/product/6?version=1
{
  "name":"hello2",
  "age":15
}

```
3. 外部(external)version

```curl
PUT shop/product/9?version=2&version_type=external
{
  "name":"test01",
  "age":12
}

```
